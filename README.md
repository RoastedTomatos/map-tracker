# Map Tracker

Застосунок реалізований на **TypeScript**, **React**, **MobX** та **Material UI**.  
Має mock-сервер, який віддає дані об’єктів для тестування.

---

1. Кожен об’єкт має:
   - унікальний ідентифікатор (ID)
   - координати (x, y або широта/довгота)
   - напрям руху (heading)

2. Якщо сервер перестає віддавати дані по об’єкту:
   - об’єкт помічається як **втраченний**
   - через **5 хвилин** він автоматично прибирається з мапи

3. Авторизація відбувається по **унікальному ключу**.
   Для тестування це ["test", "dev"]

4. Використані технології:
   - **Frontend:** React + TypeScript + MobX + Material UI + Leaflet
   - **Backend:** Node.js / Express (mock-сервер)
   - **Docker:** для запуску фронтенду та бекенду

---

## Запуск проекту локально

1. Клонуємо репозиторій:

```bash
cd map-tracker
```

2. Запуск через Docker Compose:

```bash
docker compose up --build
```

- **Frontend:** http://localhost:3000
- **Backend:** http://localhost:4000

---

## Використання

1. Відкрити фронтенд на `http://localhost:3000`
2. Авторизуватись за тестовим ключем (`demo` або `test`)
3. Мапа відобразить об’єкти, які рухаються
4. Є список втрачених об'єктів з координатами та часом, коли об'єкт був втрачений

---

## Технології

- **React** + **TypeScript** — для побудови UI
- **MobX** — управління станом об’єктів
- **Material UI** — компоненти інтерфейсу
- **Leaflet** — інтерактивна мапа
- **Node.js / Express** — mock-сервер
- **Docker** — контейнеризація фронтенду та бекенду

---

## Мок-сервер

Mock-сервер генерує дані об’єктів у форматі:

```json
[
  {
    "id": "uuid-1",
    "position": { "x": 50.1, "y": 50.2 },
    "heading": 90
  },
  {
    "id": "uuid-2",
    "position": { "x": 50.3, "y": 50.5 },
    "heading": 270
  }
]
```

- Сервер оновлює позиції об’єктів кожну секунду
- Повертає список активних об’єктів
- Об’єкти, по яких немає оновлень > 1 сек, помічаються як **втрачені**
